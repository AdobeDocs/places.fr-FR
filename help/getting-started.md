## Utilisation du service Places

Pour utiliser Places Service, effectuez les tâches suivantes :

### &#x200B;1. Assurez-vous que votre utilisateur dispose des privilèges d’accès pour utiliser Places Service

Le contexte du produit Places Service est disponible pour toutes les organisations Experience Cloud.

* Vérifiez que vous êtes répertorié en tant qu’utilisateur sous les contextes de produit Service Places et Experience Platform Launch dans Admin Console.

  Pour plus d’informations, voir *Ajout d’un utilisateur à Experience Platform Launch et Places Service* dans [Questions fréquentes](/help/places-gain-access.md).


### &#x200B;2. Création de vos points d’intérêt

Les points d’intérêt sont des clôtures géographiques que vous créez pour vos organisations. Vous pouvez créer des points d’intérêt de l’une des manières suivantes :

* À l’aide de l’interface utilisateur du service Places.

  Pour plus d’informations, voir [Présentation du service Places](/help/poi-mgmt-ui/poi-mgmt-ui-overview.md).

* Utilisation des API du service Web Places.

  Pour plus d’informations, voir [API de service web](/help/web-service-api/places-web-services.md).


### &#x200B;3. Configuration de l’extension Places

Pour plus d’informations sur la configuration de l’extension Places, voir les informations suivantes :

* [Extension Places](/help/places-ext-aep-sdks/places-extension/places-extension.md)

### &#x200B;4. (Facultatif) Implémentation d’une solution de surveillance de région

Si vous prévoyez d’effectuer une surveillance active des régions, vous devrez effectuer directement l’implémentation avec le système d’exploitation. Pour plus d’informations, consultez les documents suivants :

* iOS - [Documentation CoreLocation](https://developer.apple.com/documentation/corelocation/monitoring_the_user_s_proximity_to_geographic_regions)

* Android - [Documentation sur l’emplacement](https://developer.android.com/training/location/geofencing)
